# LocalScreen Translator

LocalScreen Translator is a Windows screen translator with screenshot, selected-text, window live, and region live modes. OCR and translation run on the local PC.

- OCR: ONNX Runtime DirectML + PP-OCRv6 (automatic CPU fallback)
- Translation: local `llama-server` + HY-MT1.5
- UI: Windows tray application with Chinese and English interfaces
- License: [Apache-2.0](./LICENSE); third-party and model licenses: [NOTICE](./NOTICE)
- 中文：[README.md](./README.md)

## What is included

After setup, OCR and translation run locally.

The current **ocr** asset covers Chinese, English, and Japanese characters, but not Hangul. OCR language coverage is determined by the recognition model in that asset.

| Content | Supplied by | Location |
|---------|-------------|----------|
| Application source | Source package | `app\`, `run.py`, and setup scripts |
| ONNX OCR models | Release **ocr** asset | `runtime\ocr\` |
| HY-MT translation model | Release **models** asset | `runtime\models\` |
| CPU/CUDA llama-server | Universal Release **llama** asset | `runtime\llama\` |
| Python environment, local configuration, and convenience launcher | Generated by `setup.ps1` | `venv\`, `config.json`, `翻译.exe` |

`翻译.exe` is a convenience launcher, not a self-contained executable. It still needs `venv`, `app`, `run.py`, and `runtime` beside it.

## Common requirements

The target PC requires:

- 64-bit Windows 10 or 11;
- Python 3.11, 3.12, or 3.13 — **Python 3.12** is recommended; do not use 3.14;
- a DirectX 12-capable GPU can accelerate OCR through DirectML; otherwise OCR uses the CPU automatically;
- approximately 4 GB of free disk space; the completed clean installation targets no more than 3 GB;
- Internet access during the first Python dependency installation.

#### Select Python 3.12 explicitly

```powershell
.\setup.ps1 -Python "C:\Path\To\Python312\python.exe"
```

To install Python first:

```powershell
winget install Python.Python.3.12
```

## Quick start

The Release **llama** asset contains both CPU and NVIDIA CUDA backends. PCs without an NVIDIA GPU use the same asset; compatible GPUs do not require a separate CUDA Toolkit installation.

### 1. Prepare the files

1. Download and extract the source package.
2. Download the **ocr**, **models**, and **llama** archives from Releases.
3. Extract all three archives into the project's `runtime\` directory. Their top-level folders must be `ocr\`, `models\`, and `llama\`, respectively; allow your archive tool to merge or replace files if prompted.

The final layout must be:

```text
project-folder\
  run.py
  翻译.exe
  app\
  runtime\
    models\
      HY-MT1.5-1.8B-Q4_K_M.gguf
    llama\
      llama-server.exe
      *.dll
    ocr\
      manifest.json
      det.onnx
      rec.onnx
      characters.txt
```

### 2. Install and check

Open PowerShell in the project root:

```powershell
# Only if Windows reports that the script is unsigned; affects this window only:
Set-ExecutionPolicy -Scope Process Bypass -Force

.\setup.ps1
.\setup.ps1 -Check
```

`setup.ps1` creates `venv`, installs dependencies, checks dependency consistency, generates the local `config.json`, selects the GPU configuration automatically, and builds `翻译.exe` when it is missing.

### 3. Run

Double-click `翻译.exe`, or run:

```powershell
venv\Scripts\pythonw.exe run.py
```

### CPU users

Auto uses the CPU when CUDA is unavailable, with no additional download. You can also open Settings → Advanced → Translation device, select CPU, and restart the app. OCR likewise falls back to the CPU when DirectML is unavailable; CPU translation is usually slower.

## Switch translation models

The app does not download additional models automatically. To switch models:

1. Download a translation `.gguf` compatible with `llama.cpp` yourself.
2. Put the model file directly in `runtime\models\`; do not add another folder level.
3. Open Tray → Settings → Advanced → Model & generation, select the model, and save.
4. Exit and restart the app before the new model takes effect.

The model directory is rescanned whenever Settings opens. Only `.gguf` files with a valid file header are listed. Translation suitability, prompt compatibility, and memory requirements depend on the selected model.

## Default hotkeys

| Action | Key |
|--------|-----|
| Screenshot translation | Alt+Q |
| Selected-text translation | Alt+W |
| Window live translation | Alt+E |
| Region live translation | Alt+R |

Use the tray menu for settings, history, and logs. Only one window or region live-translation session can run at a time.

The app opens Settings at startup and shows OCR/translation-model readiness with retry for failed items. Continuous translation can be paused and resumed without selecting the target again. Up to 50 history entries are stored as plain text in `data.db`; the History window supports search, copy, single-entry deletion, and clearing all entries. Settings, History, Translate, and subtitle geometry are restored when still visible on a connected screen.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Runtime files are missing | Extract the Release **ocr**, **models**, and **llama** archives into the project's `runtime\` directory |
| DirectML is unavailable | OCR falls back to CPU automatically; set `ocr_provider` to `cpu` in `config.json` to force it |
| GPU/llama startup fails | Select CPU under Settings → Advanced → Translation device, then restart the app |
| Annotation text is missing from screenshots/recordings | Excluded by default for speed; enable "Show annotation text in screenshots / recordings" under Settings → General (region annotation gets slightly slower) |
| `翻译.exe` is missing after setup | Run `.\setup.ps1 -BuildLauncher` to force a build, or start with `venv\Scripts\pythonw.exe run.py` |
| The app says it is already running | Check the system tray; only one instance is allowed |
| You want to verify all runtime files | Run `.\setup.ps1 -Check` |

Log file: `app.log`. See the Settings window and [SETTINGS.en.md](./SETTINGS.en.md) for advanced options.

For the runtime directory layout, see [runtime/README.en.md](./runtime/README.en.md).
