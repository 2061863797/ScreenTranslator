# ScreenTranslator

A local Windows screen translator with screenshot, selected-text, window live, region live, and screenshot OCR modes.

- OCR: PaddleOCR
- Translation: local `llama-server` + HY-MT1.5
- UI: Windows tray application with Chinese and English interfaces
- License: [Apache-2.0](./LICENSE); third-party and model licenses: [NOTICE](./NOTICE)
- 中文：[README.md](./README.md)

## What is included

After setup, OCR and translation run locally.

| Content | Supplied by | Location |
|---------|-------------|----------|
| Application source | Source package | `app\`, `run.py`, and setup scripts |
| PaddleX OCR models | Release **paddlex** asset | `runtime\paddlex\` |
| HY-MT translation model | Release **models** asset | `runtime\models\` |
| CUDA llama-server | Release **llama** asset | `runtime\llama\` |
| Python environment, local configuration, and convenience launcher | Generated by `setup.ps1` | `venv\`, `config.json`, `翻译.exe` |

`翻译.exe` is a convenience launcher, not a self-contained executable. It still needs `venv`, `app`, `run.py`, and `runtime` beside it.

## Common requirements

Every route below requires:

- 64-bit Windows 10 or 11;
- Python 3.11, 3.12, or 3.13 — **Python 3.12** is recommended; do not use 3.14;
- at least approximately 12 GB of free disk space; 15 GB is recommended during initial setup;
- Internet access during the first Python dependency installation.

#### Select Python 3.12 explicitly

```powershell
.\setup.ps1 -Python "C:\Path\To\Python312\python.exe"
```

To install Python first:

```powershell
winget install Python.Python.3.12
```

## Quick start

Choose one of the following routes based on the target PC.

### Route A: use the complete Release setup (recommended)

In addition to the common requirements, this route needs:

- an NVIDIA GPU with a working `nvidia-smi` and a driver capable of CUDA 13;

The Release **llama** asset is the NVIDIA CUDA 13 build used by this project. Its CUDA runtime DLLs are included, so a separate CUDA Toolkit installation is not required.

#### 1. Prepare the files

1. Download and extract the source package.
2. Download the **paddlex**, **models**, and **llama** archives from Releases.
3. Extract all three archives into the project's `runtime\` directory. Their top-level folders must be `paddlex\`, `models\`, and `llama\`, respectively; allow your archive tool to merge or replace files if prompted.

The final layout must be:

```text
ScreenTranslator\
  run.py
  翻译.exe
  app\
  runtime\
    models\
      HY-MT1.5-1.8B-Q4_K_M.gguf
    llama\
      llama-server.exe
      *.dll
    paddlex\
      official_models\
        PP-OCRv6_medium_det\
        PP-OCRv6_medium_rec\
```

#### 2. Install and check

Open PowerShell in the project root:

```powershell
# Only if Windows reports that the script is unsigned; affects this window only:
Set-ExecutionPolicy -Scope Process Bypass -Force

.\setup.ps1
.\setup.ps1 -Check
```

`setup.ps1` creates `venv`, installs dependencies, checks dependency consistency, generates the local `config.json`, selects the GPU configuration automatically, and builds `翻译.exe` when it is missing.

#### 3. Run

Double-click `翻译.exe`, or run:

```powershell
venv\Scripts\pythonw.exe run.py
```

### Route B: the PC does not meet the default requirements

| Situation | What to do |
|-----------|------------|
| No NVIDIA GPU | Use the **paddlex** and **models** assets, then let the script download a CPU llama; do not use the default CUDA **llama** asset |
| NVIDIA driver or CUDA llama fails | Switch to CPU mode and the CPU llama build |
| One or more Release assets are unavailable | Let the setup script download the missing runtime files from their official sources |
| Python is missing, or only Python 3.14 is installed | Install Python 3.12, then run setup again |
| Multiple Python versions are installed | Use `-Python` with the full path to Python 3.11–3.13 |

#### No NVIDIA GPU: one-command CPU setup

Download the source package and the Release **paddlex** and **models** assets. Extract both assets into `runtime\`, then run:

```powershell
.\scripts\download_runtime.ps1 -CpuOnly -SkipModel -Force
.\setup.ps1 -CpuOnly
.\setup.ps1 -Check
```

CPU mode works, but OCR and translation are considerably slower than on an NVIDIA GPU.

If the Release assets are unavailable, you can instead let the script download the required files:

```powershell
.\setup.ps1 -CpuOnly -DownloadRuntime
.\setup.ps1 -Check
```

#### NVIDIA GPU, but missing Release assets

```powershell
.\setup.ps1 -DownloadRuntime
.\setup.ps1 -Check
```

The script detects NVIDIA automatically and falls back to a CPU configuration when no NVIDIA GPU is found.

## Switch translation models

The app does not download additional models automatically. To switch models:

1. Download a translation `.gguf` compatible with `llama.cpp` yourself.
2. Put the model file directly in `runtime\models\`; do not add another folder level.
3. Open Tray → Settings → Advanced → Model & generation, select the model, and save.
4. Exit and restart the app before the new model takes effect.

The model directory is rescanned whenever Settings opens. Only `.gguf` files with a valid file header are listed. Translation suitability, prompt compatibility, and memory requirements depend on the selected model.

## Default hotkeys

| Action | Key |
|--------|-----|
| Screenshot translation | Alt+Q |
| Selected-text translation | Alt+W |
| Window live translation | Alt+E |
| Region live translation | Alt+R |
| Screenshot OCR | Alt+S |

Use the tray menu for settings, history, and logs. Only one window or region live-translation session can run at a time.

Up to 50 history entries are stored as plain text in `data.db` by default. Disable new records under Settings → General, or clear them from the History window after confirmation.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Runtime files are missing | Extract the Release **paddlex**, **models**, and **llama** archives into the project's `runtime\` directory, or run `.\setup.ps1 -DownloadRuntime` |
| Paddle installation fails | Use Python 3.12; if needed, try `.\setup.ps1 -CpuOnly` |
| GPU/llama startup fails | Update the NVIDIA driver or switch to the CPU route |
| `翻译.exe` is missing after setup | Run `.\setup.ps1 -BuildLauncher` to force a build, or start with `venv\Scripts\pythonw.exe run.py` |
| The app says it is already running | Check the system tray; only one instance is allowed |
| You want to verify all runtime files | Run `.\setup.ps1 -Check` |

Log file: `app.log`. See the Settings window and [SETTINGS.en.md](./SETTINGS.en.md) for advanced options.

For the runtime directory layout, see [runtime/README.en.md](./runtime/README.en.md).
