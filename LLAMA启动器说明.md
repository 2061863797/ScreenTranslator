# llama 启动器（玩玩用）

项目根目录的 **`启动llama.bat`** 会打开自带的图形启动器：

`runtime\llama\llama启动.exe`

这是 **单独启动本机 llama-server** 的工具，方便你手调参数、测模型，**不是** ScreenTranslator 托盘主程序。  
日常用翻译软件请双击 **`翻译.exe`** 或运行 `venv\Scripts\pythonw.exe run.py`。

---

## 怎么开

1. 双击项目根目录 **`启动llama.bat`**  
2. 或进入 `runtime\llama\` 再双击 **`llama启动.exe`**

> 必须在 `runtime\llama\` 下运行 exe（和 DLL 同目录），否则会缺库。  
> 根目录 bat 已自动 `cd` 到正确目录，推荐用 bat。

## 界面说明

### 模型文件
| 项 | 说明 |
|----|------|
| 选择模型文件 | 选 `.gguf`，例如项目里的 `runtime\models\HY-MT1.5-1.8B-Q4_K_M.gguf` |

### 核心性能
| 项 | 对应参数 | 说明 |
|----|----------|------|
| GPU层数 (-ngl) | `-ngl` | 卸到 GPU 的层数；无独显改 **0** |
| CPU线程数 (-t) | `-t` | 线程数，可按 CPU 核数调 |

### 上下文与批处理
| 项 | 参数 | 说明 |
|----|------|------|
| 上下文长度 (-c) | `-c` | 上下文 token；越大越占显存 |
| 批处理大小 (-b) | `-b` | 批大小 |
| 物理批处理 (-ub) | `-ub` | 物理批 |

### 生成参数
| 项 | 参数 | 说明 |
|----|------|------|
| 温度 (temp) | `--temp` | 越高越随机；翻译建议偏低（如 0.1～0.3） |
| Top-P / Top-K / Min-P | 采样 | 控制采样范围 |

### 功能与开关
| 项 | 说明 |
|----|------|
| 推理模式 | 部分模型的思考/推理相关开关 |
| 锁定内存 (mlock) | 锁定模型内存，减少换页 |
| K/V 缓存压缩 | KV cache 量化类选项（省显存） |

### 网络设置
| 项 | 说明 |
|----|------|
| 端口 | 默认 **8080**（与 ScreenTranslator 默认一致） |
| 主机地址 | `127.0.0.1` 仅本机；`0.0.0.0` 局域网可连 |

点 **「确认并启动」** 后启动 `llama-server`。

参数会记在同目录 **`launcher_config.json`**，下次打开会带出上次值。

## 和 ScreenTranslator 的关系

| | ScreenTranslator | 本启动器 |
|--|------------------|----------|
| 用途 | 截屏/划词/窗口翻译整套 | 只起 llama-server，手调参数 |
| 配置 | 根目录 `config.json` | `runtime\llama\launcher_config.json` |
| 端口 | 默认 8080 | 建议也 8080，两边一致才能对接 |

注意：

1. **不要同时开两个**占同一端口的 server（启动器开一个 + 软件内部再起一个会冲突）。  
2. 若你已用启动器起好 server，再开 ScreenTranslator 时，只要健康检查通过，一般会**复用**已有服务（不重复拉起）。  
3. 翻译场景温度建议偏低；启动器默认可能偏「闲聊」参数，可按喜好改。

## 推荐自测路径（玩玩）

1. 双击根目录 `启动llama.bat`  
2. 「选择模型文件」选：  
   `（项目根）\runtime\models\HY-MT1.5-1.8B-Q4_K_M.gguf`  
   （若在启动器里用相对路径，从 `llama` 目录看是 `..\models\...`）  
3. 有显卡：GPU 层 **99**；没有：关 GPU 层 / 填 **0**  
4. 端口 **8080**；主机建议 **`127.0.0.1`**（仅本机；`0.0.0.0` 会监听局域网）  
5. 确认并启动 → 浏览器打开 `http://127.0.0.1:8080` 看是否就绪  
6. 再开 ScreenTranslator（`翻译.exe` / `run.py`）做截屏/划词  

> 翻译软件自己也会在需要时启动 llama-server。若本启动器已占用 8080，软件应复用；若冲突，先关掉其中一个。
